# Paw:Paw 통합 기획 스펙 (v2 · 최종)

> 이전 대화에서 나온 모든 결정사항을 통합한 최종본입니다. `docs/features/*.md`는 이 문서를
> 기능별로 더 상세히 풀어놓은 것이므로, 작업 시 두 문서를 함께 참고하세요.

## 1. 프로젝트 성격
- 반려동물 건강관리 웹 서비스, **포트폴리오/시연용** (실사용 서비스 아님)
- AI 챗봇은 학교 제공 HAI-GPT API를 사용하며, 월간 토큰 할당량 제한(개발기 5,000 / 개강 후 20,000)이 설계 전반에 영향을 줌

## 2. 확정된 기술 스택
| 분류 | 기술 |
|---|---|
| Frontend | React (Vite) + Tailwind CSS v4 |
| Backend | FastAPI (Python) |
| DB | MongoDB Atlas |
| 인증 | Kakao 소셜 로그인 + JWT 세션 |
| AI | HAI-GPT (ChatGPT 계열, 학교 발급 키) |
| 알림 | 카카오톡 알림톡 (복약·예방접종만) |
| 배포 | Vercel / Railway |

## 3. 스코프 확정 (제외된 것들)
- ❌ 커뮤니티 기능 (자랑 피드/게시판) — 완전 제외
- ❌ 회원가입 모달 — 반려인 정보는 카카오 로그인으로만 수집
- ❌ 반려동물 추가/설정 실시간 미리보기 패널 — 제외
- ❌ 개발자 로그 버튼 — 프로덕션 화면에서 제거 (보안 이슈)

## 4. 온보딩 흐름
0. 비로그인 상태로 `/` 접속 → **인트로 페이지** 노출 (서비스 소개 + 카카오 로그인, `docs/features/intro.md`)
1. Kakao 로그인 (반려인 정보는 카카오에서 받아옴, 별도 회원가입 폼 없음)
2. 로그인 성공 → **메인페이지**로 리다이렉트. 반려동물이 없으면 빈 상태(등록 유도 버튼)만 노출
3. 반려동물 **기본정보**(이름·나이·종)는 사용자가 직접 입력
4. 반려동물 **세부정보**(복약·예방접종)는 AI 챗봇과의 대화를 통해 등록 — 6장 참고

## 5. 화면 구성
- **인트로 페이지** (신규 추가): 로그인 전 첫 진입 화면
- NAV(로그인 후 공통): 대시보드 / AI챗봇 / 건강 레포트 / 마이펫이지
- **메인페이지**(로그인 후 첫 화면, 간소화 대시보드): 중앙 마스코트 애니메이션 자리 + 알림 우선순위 기반 말풍선 + 좌측 축소 위젯. 상세는 `docs/features/dashboard.md` 참고
- 전체 대시보드, AI 챗봇, 건강 레포트, 마이펫이지, 반려동물 추가/설정
- 세부 화면별 스펙은 `docs/features/*.md` 참고 (신규: `intro.md`)

## 6. AI 챗봇 아키텍처 (핵심 설계)

### 6.1 두 개의 병렬 감지기
챗봇은 "모드 전환"이 아니라, 사용자의 모든 메시지가 **두 감지기를 동시에 통과**하는 구조로 동작한다.

| 감지기 | 항상 동작 | LLM 호출 필요 여부 | 결과 |
|---|---|---|---|
| 증상 감지기 | O | O (자유 상담 답변 생성 시 같이 처리) | '뽀삐의 건강'에 병명 없이 조용히 기록 |
| 등록 감지기 (복약/예방접종) | O | ✕ (키워드/정규식 매칭, 토큰 소모 없음) | "등록할까요?" 제안 → 슬롯필링 시작 |

### 6.2 등록 슬롯필링 흐름
자유 텍스트 파싱 대신 **버튼/피커 기반 단계별 질문**으로 데이터를 채운다 (오답 가능성 최소화, LLM 호출 없음).

1. 반려동물 선택 (버튼)
2. 항목별 질문 — 아래 두 패턴 중 하나
3. 요약 카드 확인 (확인/수정)
4. 저장 → 리스트 모달에서 언제든 재수정 가능

### 6.3 등록 데이터 스키마 (두 패턴으로 일반화)
카카오 알림 대상은 복약·예방접종 두 가지뿐이며, 실제로는 아래 2개 패턴으로 수렴한다.

| 패턴 | 대상 | 필요 입력 | 리마인드 |
|---|---|---|---|
| 반복형 | 복약 | 반려동물, 약이름, 하루 횟수(1~5), 알림시간(최대 3회) | 매일 지정 시간 |
| 단일날짜형 | 예방접종 | 반려동물, 항목명, 날짜 | 1주전·1일전·당일 (고정값, 사용자가 정하지 않음) |

```json
{
  "petId": "string",
  "type": "medication | single_date",
  "title": "string",
  "frequencyPerDay": "number (medication만)",
  "alertTimes": ["HH:mm", "... (medication만, 최대 3개)"],
  "date": "YYYY-MM-DD (single_date만)"
}
```

### 6.4 병명 표기 규칙 (매우 중요)
- 챗봇이 사용자에게 주는 **답변 자체**: 병명 언급 가능 ("관절염일지도 모릅니다", 통계·출처 포함)
- **대시보드에 자동 업로드되는 기록**: 병명 절대 금지 ("다리 절뚝임(시작: 3일전)" 수준까지만)
- API 필드로는 `symptomSummary`(병명 금지, 대시보드 저장용) / `suspectedSymptom`(병명 가능, 요약 모달 전용)로 분리

## 7. 토큰 절약 설계 원칙
1. **등록 감지기는 LLM을 쓰지 않는다** — 키워드/정규식 매칭으로 처리
2. **슬롯필링 UI는 텍스트 생성이 필요 없다** — 버튼/피커로만 구성
3. LLM 호출은 **자유 상담 답변 생성**에만 사용 (답변 + symptomSummary + suspectedSymptom을 한 번의 호출로 같이 받기 — 호출 횟수 최소화)
4. 프롬프트에 대시보드 데이터 전체를 넣지 않고, 최근 체중/최근 특이사항 등 요약본만 전달
5. `max_tokens` 캡을 걸어 응답 길이 제한
6. 개발 기간(월 5,000토큰)에는 목업 응답 모드로 개발하고, 실제 API 호출은 프롬프트 검증 시에만 최소한으로 사용
7. 발표/시연 날에는 실시간 호출 대신 사전에 캐시해둔 응답을 재생하는 방식을 권장
8. 할당량 소진 시 "AI 상담이 일시적으로 제한됩니다" 같은 안내로 우아하게 처리 (에러로 죽지 않게)

## 8. 카카오톡 채널 연동
- 복약 알림: 지정 시간에 알림톡 발송, 말풍선 내 [복용 완료] 버튼 클릭 시 웹 접속 없이 DB 실시간 반영 (`POST /api/v1/callback/kakao-medication`)
- 예방접종 D-Day: 1주전/1일전/당일 자동 리마인드
